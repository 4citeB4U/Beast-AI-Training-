# Module 6: One-Click Sovereignty

The ultimate goal of the Sovereign Architect is to build systems that exist entirely within a controlled, independent environment. We must move away from fragile, multi-cloud setups and embrace unified deployment.

## The Four-Tier Stack
As detailed in the Sovereign Architect’s Handbook, the Sovereign ecosystem relies on a strict four-tier stack managed by `ExecutionEngine.ts`.

1. **The Brain (FastAPI):** Python-based orchestration handling the 8 stages of the Sovereign Cycle.
2. **The Interface (React/Gradio):** The visual "Aura" layer where users interact with the system.
3. **The Memory (ONNX Vector Engine):** Local, cloud-free embedding database ensuring privacy and fast retrieval for Episodic and Semantic memory.
4. **The Executors (PowerShell/Node):** Background processes running the Compliance Fleet and local file operations.

## Docker Orchestration
Sovereignty means portability. The entire ecosystem is deployed using `Docker Compose`.

### Why Docker Compose?
*   **Isolation:** The Vector Engine runs in its own secure container, accessible only by the Nexus routing agents.
*   **One-Click Launch:** Users and architects can spin up the entire Society of Agents using a single `docker-compose up` command.
*   **Self-Healing:** If an agent container crashes (e.g., Nova gets stuck in an infinite loop), Docker restarts it without breaking the overall system, maintaining the "Repairable" standard.
