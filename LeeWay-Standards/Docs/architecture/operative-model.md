# LeeWay Operative Model

## Canonical Definition

> A LeeWay Operative is a single-instance, endpoint-bound workforce unit dispatched by an Agent to a specific device or interface surface, where it performs real operational tasks through MCP, protocol-level integrations, and governed system controls.

## Agent vs Operative Separation

### Agents

- Think
- Plan
- Route
- Govern

### Operatives

- Execute
- Interact with endpoint surfaces
- Return results and receipts to the system

**Physical model:**

> An Agent lives in the system. An Operative lives on the endpoint.

## Five Hard Conditions

1. Endpoint-bound: one operative is attached to one active endpoint surface.
2. Single-instance: one operative cannot self-duplicate within one assignment context.
3. Execution-grade: operative duty is real work, not abstract chat.
4. Protocol-enabled: actions occur through approved channels only.
5. Non-human efficiency: consistency comes from software execution, not biological endurance.

## Endpoint Classes

| Class | Primary Surface | Typical Scope | Primary Channels |
|---|---|---|---|
| Mobile Operative | Phone and tablet UI surfaces | App navigation, forms, field workflows, mobile notifications | Mobile runtime controls, approved app protocols, MCP portals |
| Desktop Operative | Workstation and desktop app/browser surfaces | Office workflows, dashboards, file workflows, enterprise portals | Browser/runtime controls, desktop commander MCP, approved integrations |
| Communications Operative | Call, messaging, and communication interfaces | Calling, drafting and sending messages, communication queue handling | RTC channels, voice pipeline, messaging connectors, approved service APIs |

## Hard Operational Rules

- One operative occupies only one active endpoint at a time.
- One operative has one live assignment context.
- No uncontrolled self-duplication is permitted.
- Operatives execute only through approved tools, protocols, and governance gates.
- Operatives return execution artifacts, results, and status to the system.
- Agents may dispatch multiple operatives, but each operative remains individually endpoint-bound.

## Execution Channels

Operatives may execute through:

- MCP servers
- Device bridges
- App and service protocols
- RTC channels
- Browser/runtime automation controls
- Approved integration hooks

## Efficiency Statement

> Operatives outperform human workers in repetitive and interface-driven tasks because they are not limited by fatigue, travel, or biological maintenance. Their advantage comes from protocol-level action, persistent focus, and continuous system memory/tooling access.
