# LeeWay Core Sovereignty Doctrine

Status: ratified architectural law
Effective date: 2026-04-17
Owner layer: LeeWay-Standards

## 1) Foundational law

Nothing in the LeeWay Product Line exists independently.

Everything is born from, governed by, and traceable to two sovereign core layers:

1. LeeWay-Standards
2. LeeWay-Edge-Integrated

No repository, runtime, agent family, or product surface is sovereign outside this core pair.

## 2) Sovereign core layer duties

### 2.1 LeeWay-Standards (first cause)

LeeWay-Standards owns:

1. law and governance policy
2. permissions and regulatory controls
3. trust model and evidence requirements
4. agent identity and birth rules
5. audit and chain-of-evidence policy
6. corrective permanence logic
7. shared learning pipeline contracts
8. canonical critical solution patterns

Hard law: all system agents are born from LeeWay-Standards.

### 2.2 LeeWay-Edge-Integrated (motherboard and embodiment)

LeeWay-Edge-Integrated owns:

1. motherboard orchestration
2. render orchestration for all user-visible surfaces
3. construct embodiment (also called Room on the Edge)
4. session and runtime orchestration interfaces
5. user-facing presentation of agent civilization and product line

Hard law: all user-visible LeeWay experience is rendered through LeeWay-Edge-Integrated.

## 3) Subordinate runtime layers

RTC and GPU layers are operationally critical but not sovereign.

1. RTC provides communication, signaling, presence, and continuity.
2. GPU provides compute acceleration and heavy processing.

They operate under Standards law and Integrated orchestration.

Required authority chain:

LeeWay-Standards -> LeeWay-Edge-Integrated -> runtime layers -> projections

## 4) Projection sovereignty prohibition

Projection repos (including Agent Lee OS, World Within, employee-facing systems, and future product surfaces) are projections and adapters, not sovereign law centers.

Allowed in projection repos:

1. echo files
2. adapter files
3. placement and tracing files
4. render surfaces and local presentation composition

Not allowed in projection repos:

1. independent critical governance logic
2. forked core contracts as source-of-truth
3. private runtime internals that bypass core chain

## 5) Echo file doctrine

Echo files are required where placement and traceability require familiar filenames in projection repos.

Echo file requirements:

1. same recognizable filename is allowed
2. file must declare canonical core source path
3. file must identify owner layer and projection layer
4. file must link or adapt to core logic, not replace it
5. file must include audit trace metadata
6. file must never become a second source-of-truth

## 6) System-agent birth law

Every critical solution must complete this lifecycle:

1. issue discovered
2. issue analyzed
3. issue solved
4. solution standardized in LeeWay-Standards
5. standards-born system agent assigned permanent monitoring duty

No critical solution is complete until a Standards-born maintenance agent owns the issue class.

## 7) Hive mind specialization law

Agents may be specialized by role and surface, but no specialization creates civilizational separation.

All agents share:

1. one trust fabric
2. one evidence model
3. one regulatory universe
4. one shared learning pipeline

Specialization creates order, not isolation.

## 8) Trust and evidence law

The trust fabric is 100 percent evidence-bound.

Required controls:

1. every action is exposed
2. every action is logged
3. every action is auditable
4. parallel audits are expected
5. chain-of-evidence is mandatory

No agent acts without trace. No trace exists without audit. No audit exists without Standards authority.

## 9) Construct definition

The construct (also called Room on the Edge) is the rendered environment produced through LeeWay-Edge-Integrated where users interact with Agent Lee, system agents, employee agents, and the full LeeWay product line.

Naming note: different agent families may prefer Construct or Room on the Edge; both terms refer to the same integrated rendered environment.

## 10) Non-negotiable architectural rules

1. all system agents are born from LeeWay-Standards
2. all user experience is rendered through LeeWay-Edge-Integrated
3. all critical logic lives in core layers, not projections
4. projection repos may contain echoes, never sovereign duplicates
5. every critical issue becomes a Standards-owned monitored agent responsibility
6. every agent action is exposed, logged, and auditable
7. nothing in the LeeWay Product Line exists outside the hive

## 11) Compliance evidence requirements

To claim compliance, each non-core layer must produce:

1. an echo doctrine file with canonical source pointer
2. mapping of local adapters to core owners
3. evidence of no sovereign duplicate critical logic
4. audit linkage to Standards and Integrated governance

## 12) Operational consequence for cleanup

Projection repos must continue hollowing toward:

1. render surfaces
2. adapters
3. placement and tracing echoes

Core logic must continue converging into:

1. LeeWay-Standards
2. LeeWay-Edge-Integrated
3. approved runtime layers under core authority

This doctrine is binding for new work and migration work.
